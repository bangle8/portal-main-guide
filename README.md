# portal-main-guide

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可能来自不同项目、不同用途，或是临时生成的静态内容。仓库本身不绑定任何特定域名或网站，仅作为页面文件的集中存放与版本管理场所。

## 项目简介

portal-main-guide 是一个通用型 HTML 页面归档仓库。它适用于以下场景：

- 需要将多个独立 HTML 文件集中管理
- 希望利用 GitHub Pages 或其他静态托管服务展示这些页面
- 为不同项目或实验提供统一的页面输出位置
- 作为临时页面或演示页面的归档备份

仓库不包含后端逻辑、数据库或动态内容生成机制，所有页面均为纯静态 HTML 文件。

## 目录说明

仓库根目录下可能包含以下内容：

```
portal-main-guide/
├── index.html          # 可选的主入口页面
├── pages/              # 存放归档的 HTML 页面
│   ├── example-01.html
│   ├── example-02.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
├── README.md           # 本文件
└── .gitignore
```

- `pages/` 目录是主要归档位置，每个子目录或文件对应一个独立页面。
- `assets/` 目录存放页面可能共享的样式、脚本或图片资源。
- `index.html` 可选，用于提供一个导航或索引页面。

目录结构可能随仓库内容更新而变化，请以实际文件为准。

## 页面归档说明

每个归档的 HTML 页面应遵循以下约定：

- 页面应自包含，尽量不依赖外部资源（或明确声明依赖）。
- 页面标题、描述、元数据等信息建议包含在 HTML 头部。
- 页面命名应具有辨识度，建议使用英文小写字母、数字和连字符组合。
- 若页面引用 `assets/` 中的资源，请使用相对路径。

归档页面不保证与任何外部服务或域名关联，仅作为独立文件存在。

## 维护说明

本仓库由维护者不定期更新。维护内容包括：

- 添加新的 HTML 页面
- 更新或修正已有页面内容
- 清理不再需要的旧页面
- 优化目录结构与资源引用

欢迎通过 Issues 或 Pull Requests 提出建议或报告问题。提交新页面时，请确保文件符合归档规范，并尽量提供简洁的说明。

## 使用方式

你可以通过以下方式使用本仓库中的页面：

1. **直接浏览**：克隆仓库后，在浏览器中打开本地 HTML 文件。
2. **托管部署**：将仓库内容部署到任何静态文件托管服务（如 GitHub Pages、Netlify 等）。
3. **引用嵌入**：在需要时通过链接或 iframe 引用具体页面（需自行搭建环境）。

仓库不提供在线演示链接或访问入口，请自行决定如何部署与使用。
